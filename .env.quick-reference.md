# Environment Variables Quick Reference

## üöÄ Get Started in 30 Seconds

```bash
# 1. Copy the example file
cp .env.example .env.local

# 2. (Optional) Edit your settings
nano .env.local

# 3. Start developing
npm run dev
```

## üìù Common Configurations

### Development (Debug Mode On)
```bash
# .env.local
VITE_DEBUG_MODE=true
VITE_SHOW_FPS=true
```

### Testing with Boosted Stats
```bash
# .env.local
VITE_STARTING_LEVEL=10
VITE_STARTING_GOLD=10000
```

### Custom App Title
```bash
# .env.local
VITE_APP_TITLE=My Awesome Game [DEV]
```

## ‚ö° Most Used Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `VITE_DEBUG_MODE` | `false` | Show debug console logs |
| `VITE_SHOW_FPS` | `false` | Display FPS counter |
| `VITE_STARTING_LEVEL` | `1` | Player starting level |
| `VITE_STARTING_GOLD` | `0` | Starting gold amount |
| `VITE_APP_TITLE` | `Idle Quest...` | Browser tab title |

## üîë Golden Rules

1. ‚úÖ Always prefix with `VITE_`
2. ‚úÖ Restart dev server after changes
3. ‚úÖ Use `.env.local` for personal settings
4. ‚ùå Never commit `.env.local`
5. ‚ùå Never store API keys/secrets

## üÜò Troubleshooting

**Not working?**
- Did you restart the dev server? (`Ctrl+C` then `npm run dev`)
- Does the variable name start with `VITE_`?
- Is the file named exactly `.env.local` (no extra .txt)?

**Need more help?**
- See `ENVIRONMENT_SETUP.md` for detailed guide
- Check `.env.example` for all available variables
